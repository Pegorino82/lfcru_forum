# UC-003: Управление пользователями

## Goal

Пользователь забанен (не может входить и участвовать в форуме) или разбанен (доступ восстановлен).

## Primary Actor

Admin, Moderator

## Trigger

Admin или Moderator находит нарушителя и хочет ограничить ему доступ — или снять ограничение.

## Preconditions

- Пользователь аутентифицирован с ролью Admin или Moderator.
- Пользователь находится в admin-панели, раздел «Пользователи».
- Целевой пользователь зарегистрирован в системе.

## Main Flow

### Бан

1. Admin/Moderator открывает список пользователей, находит нужного (по email или имени).
2. Выбирает действие «Заблокировать».
3. Подтверждает. Пользователь помечается как заблокированный.
4. Заблокированный пользователь при следующей попытке входа получает отказ.
5. Активные сессии заблокированного пользователя аннулируются или отклоняются при следующем запросе.

### Разбан

1. Admin/Moderator находит пользователя в списке заблокированных.
2. Выбирает «Разблокировать». Блокировка снимается.
3. Пользователь снова может входить в систему.

## Alternate Flows / Exceptions

- `ALT-01` Admin/Moderator пытается заблокировать другого Admin — поведение определяется в feature.md (OQ-* или явное правило).
- `EX-01` Целевой пользователь уже заблокирован — действие идемпотентно, возвращается успех или информационное сообщение.

## Postconditions

- После бана: поле `banned_at` (или флаг `is_banned`) в таблице `users` обновлено; пользователь не может войти.
- После разбана: блокировка снята; пользователь снова может войти.

## Business Rules

- `BR-01` Только Admin и Moderator могут банить/разбанивать пользователей.
- `BR-02` Банить самого себя нельзя.
- `BR-03` История банов (кто, когда, за что) — вне scope этого UC.
