# UC-002: Управление структурой форума

## Goal

Структура форума обновлена: создан/отредактирован раздел или тема, изменения отражаются в публичном форуме.

## Primary Actor

Admin, Moderator

## Trigger

Admin или Moderator хочет добавить новый раздел форума или новую тему в существующий раздел.

## Preconditions

- Пользователь аутентифицирован с ролью Admin или Moderator.
- Пользователь находится в admin-панели.
- Для создания темы: целевой раздел уже существует.

## Main Flow

### Создание раздела

1. Пользователь открывает список разделов в admin-панели.
2. Указывает название и (опционально) описание раздела.
3. Сохраняет. Раздел появляется в публичном форуме.

### Создание темы

1. Пользователь выбирает раздел.
2. Указывает название темы.
3. Сохраняет. Тема появляется в выбранном разделе.

### Редактирование

1. Пользователь выбирает раздел или тему из списка.
2. Изменяет название/описание.
3. Сохраняет. Изменения отражаются немедленно.

## Alternate Flows / Exceptions

- `ALT-01` Попытка создать раздел с пустым названием — форма не отправляется, отображается ошибка валидации.
- `EX-01` Конфликт имён разделов — возвращается понятная ошибка (если уникальность нужна, фиксируется в feature.md).

## Postconditions

- Раздел/тема существуют в БД и видны в публичном форуме.
- Существующие темы и сообщения в разделе не затрагиваются при редактировании названия раздела.

## Business Rules

- `BR-01` Только Admin и Moderator могут создавать и редактировать разделы и темы.
- `BR-02` Удаление разделов/тем — вне scope этого UC (требует отдельного решения по orphaned content).
