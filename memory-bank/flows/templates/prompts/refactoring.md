Тип рефакторинга (выбери один):
- по ходу delivery — делай в рамках текущей задачи, минимальный scope;
- исследовательский — сначала опиши что нашёл, потом согласуй план;
- системный — обязателен явный план с checkpoints до начала изменений.

Правила:
- Не меняй поведение — только структуру.
- Не расширяй scope без явного согласования.
- После каждого логического шага — тесты зелёные (Go на хосте не установлен — Docker-команды в `memory-bank/engineering/testing-policy.md` → Stack).
- Если находишь конфликтующие паттерны — остановись и спроси, не угадывай.
- Simplify review: нет premature abstractions, dead code, дублирования.
- Коммит: refactor: <описание>.
- Обнови HANDOFF.md.

Правила автономии:
- Покажи план перед: изменением публичных интерфейсов (Go-интерфейс, HTTP-handler, структура данных), удалением файлов/кода.
- Остановись и спроси: конфликтующие паттерны, выход за согласованный scope.
