# 04_article_page

## Brief
[GitHub Issue](https://github.com/Pegorino82/lfcru_forum/issues/4)

## Проблема

Сайт не имеет страницы просмотра статьи — при клике на ссылку в списке новостей ничего не происходит. Пользователь видит заголовок и дату, но не может прочитать полный текст.

## Для кого

- Гости сайта (основной трафик, не авторизованы)
- Зарегистрированные пользователи

## Желаемый результат

Пользователь, кликнув на новость в списке, попадает на отдельную страницу и читает полный текст статьи. Неопубликованные или несуществующие статьи недоступны.

## Acceptance Criteria

- [ ] При переходе по URL `/news/{id}` отображается страница статьи с заголовком, датой публикации и полным текстом
- [ ] Ссылки в блоке «Последние новости» на главной странице ведут на страницу соответствующей статьи
- [ ] Если статья с указанным ID не существует или не опубликована — возвращается страница 404
- [ ] Страница доступна для гостей без авторизации

## Контекст

Главная страница уже отображает список последних новостей со ссылками вида `/news/{id}`, но переход по ним ведёт в никуда — роута и страницы не существует. Это обрывает основной пользовательский сценарий: пришёл на сайт → увидел заголовок → хочет прочитать. Модель `News` с полем `Content` уже есть в коде.
