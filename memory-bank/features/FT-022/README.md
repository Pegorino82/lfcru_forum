# FT-022 — fix: некорректное отображение даты матча на главной

## Описание бага

На главной странице в блоках «Последний матч» и «Следующий матч» отображалась
некорректная дата — всегда «января», независимо от фактического месяца матча.
Дополнительно: время следующего матча подписывалось «МСК», хотя хранится в UTC.

## Условия воспроизведения

Открыть главную страницу → секция «Последний матч» и «Следующий матч».
Любой матч не в январе отображался с неверным месяцем.

## Корневая причина

В Go строка формата `"02 января 2006"` — слово «января» является **литералом**,
а не спецификатором месяца. Поэтому любая дата отображалась с «января»
вне зависимости от реального месяца.

## Решение

1. **`internal/tmpl/renderer.go`** — добавлена template-функция `ruDate`,
   которая форматирует `time.Time` с русским названием месяца
   (`fmt.Sprintf("%02d %s %d", t.Day(), ruMonths[t.Month()-1], t.Year())`).

2. **`templates/home/index.html`** — в блоках «Последний матч» и «Следующий матч»
   `.Format "02 января 2006"` заменено на `ruDate`.
   Подпись «МСК» заменена на «GMT» (MatchDate хранится в UTC).

3. **`internal/football/client.go`** — `fetchLast`: добавлен `dateFrom/dateTo`
   (окно 60 дней), берётся последний элемент ответа вместо первого.
