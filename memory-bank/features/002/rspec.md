# [APPROVED] 02_layout_base — Спецификация (SDD)

## 1. Контекст

Сайт и форум LFC.ru используют единый базовый шаблон `templates/layouts/base.html`. Сейчас он содержит хэдер (навигационная панель) с логотипом и ссылками, но футер отсутствует. При добавлении новых страниц (главная, новости, форум и т.д.) необходимо гарантировать, что каждая страница наследует единый layout с хэдером и футером.

**Ссылка на бриф:** [brief.md](brief.md)
**GitHub Issue:** [#2](https://github.com/Pegorino82/lfcru_forum/issues/2)

---

## 2. Акторы

| Актор | Описание |
|---|---|
| Гость | Незарегистрированный посетитель сайта и форума |
| Пользователь | Зарегистрированный аккаунт |
| Модератор | Наследует поведение Пользователя |
| Администратор | Наследует поведение Пользователя |

> Для данной фичи (layout) Модератор и Администратор ведут себя как Пользователь — показывается username и ссылка «Выйти».

---

## 3. Текущее состояние

Шаблон `templates/layouts/base.html` уже определяет:
- `<nav>` с логотипом «LFC.ru» и блоком `{{template "nav" .}}` (ссылки: Вход/Регистрация для гостя, username/Выйти для пользователя)
- `<div class="container">` с flash-сообщением и блоком `{{block "content" .}}`
- Подключение HTMX и Alpine.js

**Что отсутствует:**
- Футер (`<footer>`)
- Ссылки навигации на разделы сайта и форума в хэдере (пока только авторизация)

---

## 4. Сценарии использования (Use Cases)

### UC-1: Отображение хэдера на каждой странице

**Актор:** Гость / Пользователь

**Основной поток:**

1. Пользователь открывает любую страницу сайта или форума
2. Система рендерит страницу через `base.html`
3. Хэдер отображается в верхней части страницы
4. Хэдер содержит: логотип «LFC.ru» (ссылка на `/`), навигационные ссылки, блок авторизации

**Инвариант:** хэдер одинаков на всех страницах. Различие только в блоке авторизации (гость vs пользователь) — уже реализовано в `{{template "nav" .}}`.

**Acceptance Criteria:**
- **Given** пользователь открывает любую страницу, **When** страница загружена, **Then** HTML-ответ содержит `<header>` с `<nav aria-label="Основная навигация">`
- **Given** пользователь не авторизован, **When** страница загружена, **Then** хэдер содержит ссылки «Войти» (`/login`) и «Регистрация» (`/register`)
- **Given** пользователь авторизован, **When** страница загружена, **Then** хэдер содержит username пользователя и кнопку формы «Выйти»
- **Given** username длиннее 20 символов, **When** страница загружена, **Then** username усечён с `…` и полное имя доступно в `title`

### UC-2: Отображение футера на каждой странице

**Актор:** Гость / Пользователь

**Основной поток:**

1. Пользователь открывает любую страницу сайта или форума
2. Система рендерит страницу через `base.html`
3. Футер отображается в нижней части страницы
4. Футер содержит: копирайт, дисклеймер

**Инвариант:** футер одинаков на всех страницах, не зависит от авторизации.

**Acceptance Criteria:**
- **Given** пользователь открывает любую страницу, **When** страница загружена, **Then** HTML-ответ содержит `<footer>` с текстом `© 2026 LFC.ru — фан-сайт болельщиков ФК «Ливерпуль»`
- **Given** пользователь открывает любую страницу, **When** страница загружена, **Then** футер содержит текст `Не является официальным сайтом Liverpool FC`
- **Given** контента на странице мало, **When** страница загружена, **Then** футер прижат к низу viewport (sticky footer)

### UC-3: Навигация между разделами

**Актор:** Гость / Пользователь

**Основной поток:**

1. Пользователь видит навигационные ссылки в хэдере
2. Пользователь кликает по реализованной ссылке
3. Система загружает целевую страницу
4. Хэдер и футер остаются на месте, не меняют внешний вид

**Альтернативный поток (ссылка-заглушка):**
- Пользователь кликает «Форум» или «Новости» → `href="#"` → браузер не выполняет навигацию, страница остаётся прежней

**Acceptance Criteria:**
- **Given** навигационные ссылки являются заглушками, **When** пользователь кликает по ним, **Then** переход не происходит (`href="#"`)
- **Given** любая страница загружена, **When** пользователь переходит на другую страницу, **Then** `<header>` и `<footer>` структурно идентичны на обеих страницах

## 5. Состав хэдера

| Элемент | Описание | Ссылка |
|---|---|---|
| Логотип | Текст «LFC.ru» | `/` |
| Навигация (левая часть) | Ссылки на основные разделы | см. ниже |
| Авторизация (правая часть) | Вход/Регистрация или username/Выйти | уже реализовано |

### Навигационные ссылки

На данном этапе ссылки являются заглушками. **`href` заглушек = `"#"`** — браузер не выполняет навигацию. По мере реализации разделов `href` заменяется на реальный URL.

| Раздел | href сейчас | href после реализации |
|---|---|---|
| Форум | `#` | `/forum` |
| Новости | `#` | `/news` |

> **Примечание:** Минимальный набор ссылок для MVP. Трансферы, таблица, календарь, архив — добавляются по мере реализации.

### Блок авторизации

| Состояние | Содержимое |
|---|---|
| Гость | Ссылки «Войти» (`/login`) и «Регистрация» (`/register`) |
| Пользователь | Username (макс. 20 символов, усечение + `title` при превышении) + `<form method="POST" action="/logout">` с кнопкой «Выйти» и CSRF-токеном |

> Logout реализован через `<form method="POST">`, а не `<a>` — для CSRF-защиты и семантики.

### Active state навигации

На данном этапе active state **не реализуется** — все ссылки являются заглушками. При реализации конкретных разделов в будущих фичах добавить `aria-current="page"` для активной ссылки и CSS-класс `nav-link--active`.

## 7. Изменения в шаблонах

### `templates/layouts/base.html`

Изменения в существующем файле:

1. **Обернуть `<nav>` в `<header>`** — для семантики landmark-региона и корректной работы screen readers.

2. **Добавить skip-link** перед `<header>` — ссылка «Перейти к содержимому» с `href="#content"`.

3. **Хэдер — добавить навигационные ссылки.** Между логотипом и блоком авторизации добавить ссылки на разделы (Форум, Новости).

4. **Добавить `id="content"` к `.container`** — цель для skip-link и HTMX-swap.

5. **Футер — добавить `<footer>` после `.container`.** Содержимое: копирайт и дисклеймер.

6. **Sticky footer.** Футер прижат к низу viewport. Реализация: `body` как flex-контейнер с `min-height: 100vh`, `.container` получает `flex: 1`.

### Структура HTML (целевая)

```html
<body>
  <a href="#content" class="skip-link">Перейти к содержимому</a>
  <header>
    <nav aria-label="Основная навигация">
      <a href="/" class="logo">LFC.ru</a>
      <div class="main-nav">
        <a href="#" class="nav-link">Форум</a>
        <a href="#" class="nav-link">Новости</a>
      </div>
      <div class="nav-auth">
        {{if .User}}
          <span class="nav-username"
                title="{{.User.Name}}"
          >{{truncate .User.Name 20}}</span>
          <form method="POST" action="/logout">
            <input type="hidden" name="_csrf" value="{{.CSRFToken}}">
            <button type="submit">Выйти</button>
          </form>
        {{else}}
          <a href="/login">Войти</a>
          <a href="/register">Регистрация</a>
        {{end}}
      </div>
    </nav>
  </header>

  <main id="content" class="container" tabindex="-1">
    {{template "flash" .}}
    {{block "content" .}}{{end}}
  </main>

  <footer>
    <p>© 2026 LFC.ru — фан-сайт болельщиков ФК «Ливерпуль»</p>
    <p>Не является официальным сайтом Liverpool FC</p>
  </footer>
</body>
```

> `{{truncate .User.Name 20}}` — вспомогательная функция шаблона (template func), усекает строку до 20 символов с добавлением `…`. Если функция не добавлена, можно реализовать через Go-обёртку или хранить в модели усечённое имя.

Минимальная стилизация (inline `<style>` в `base.html`, как уже принято в проекте):

| `footer a` | `color: #fff;` |
| `.main-nav` | `display: flex; gap: 1rem;` |

| `.skip-link` | `position: absolute; top: -40px; left: 0; background: #c8102e; color: #fff; padding: 0.5rem 1rem; z-index: 100; transition: top 0.1s;` |
| `.skip-link:focus` | `top: 0;` |

### Мобильное поведение (до реализации бургер-меню)

До реализации адаптивного меню (отдельная задача) на узких экранах (< 768px) навигационные ссылки `.main-nav` скрываются:

```css
@media (max-width: 768px) {
  .main-nav { display: none; }
}
```

> Это временное решение. Логотип и блок авторизации остаются видимыми на мобильных.

## 9. HTMX-совместимость

### Поведение

При HTMX-запросах (`HX-Request: true`) сервер возвращает только фрагмент контента — без `<header>`, `<footer>` и `<html>/<body>`. Хэдер и футер не перерисовываются.

### Технический механизм

В каждом Echo-хэндлере, поддерживающем HTMX-partial render:

```go
if c.Request().Header.Get("HX-Request") == "true" {
    return c.Render(http.StatusOK, "content", data) // только блок content
}
return c.Render(http.StatusOK, "base.html", data)   // полный layout
```

> Проверка `HX-Request` — ответственность конкретного хэндлера, а не глобального middleware. Это позволяет хэндлерам явно контролировать поведение.

### Поддерживаемые маршруты (partial render)

Все маршруты, рендерящие HTML-страницы: `/`, `/login`, `/register` (и будущие `/forum`, `/news`).

### Edge cases

| Сценарий | Поведение |
|---|---|
| `HX-Request: true` + обычный `hx-get` | Возвращается только фрагмент (блок `content`) |
| `HX-Request: true` + `hx-target="body"` | Не используется в проекте; если встретится — хэндлер возвращает полный layout |
| Запрос без `HX-Request` | Всегда возвращается полный layout |

### Acceptance Criteria

- **Given** запрос с заголовком `HX-Request: true`, **When** хэндлер обрабатывает запрос, **Then** ответ не содержит `<header>`, `<footer>`, `<html>`, `<body>`
- **Given** запрос без заголовка `HX-Request`, **When** хэндлер обрабатывает запрос, **Then** ответ содержит полный layout с `<header>` и `<footer>`

## 10. Accessibility

### User Stories

- Как пользователь скринридера, я хочу пропустить навигацию через skip-link, чтобы сразу перейти к содержимому страницы
- Как пользователь клавиатурной навигации, я хочу обходить все интерактивные элементы через Tab в логичном порядке

### Требования и реализация

| Требование | Реализация | AC |
|---|---|---|
| Семантический хэдер | `<header>` оборачивает `<nav>` | HTML содержит `<header>` |
| Семантическая навигация | `<nav aria-label="Основная навигация">` | HTML содержит `aria-label="Основная навигация"` |
| Семантический футер | `<footer>` | HTML содержит `<footer>` |
| Семантический контент | `<main id="content">` вместо `<div class="container">` | HTML содержит `<main id="content">` |
| Skip-link | `<a href="#content" class="skip-link">Перейти к содержимому</a>` перед `<header>`, скрыт, виден при фокусе | HTML содержит `<a href="#content">` первым в `<body>`; при Tab — ссылка появляется |
| Клавиатурная навигация | Все интерактивные элементы доступны через Tab в порядке: skip-link → nav (логотип, ссылки, авторизация) → main → footer | Tab обходит все ссылки и кнопки |

## 11. Тест-план

### Визуальная проверка (ручная)

| Сценарий | Ожидаемый результат |
|---|---|
| Открыть `/login` (гость) | Хэдер и футер присутствуют, ссылки «Войти» и «Регистрация» видны |
| Открыть `/login` (авторизованный) | Хэдер содержит username и кнопку «Выйти» |
| Открыть `/register` | Хэдер и футер присутствуют |
| Открыть `/` (главная) | Хэдер и футер присутствуют |
| Уменьшить высоту окна (короткая страница) | Футер прижат к низу viewport |
| Страница с длинным контентом | Футер под контентом, не перекрывает |
| Ширина окна < 768px | `.main-nav` скрыт, логотип и авторизация видны |
| Username > 20 символов | Отображается усечённый текст с `…`, полное имя в `title` |
| Нажать Tab на пустой странице | Skip-link появляется первым, фокус видим |

### Интеграционные тесты

Тестируемые URL: `/`, `/login`, `/register`.

| Сценарий | URL | Ожидаемый результат |
|---|---|---|
| Полный layout — nav | GET `/login` (гость) | Ответ содержит `<header>` и `<nav` |
| Полный layout — footer | GET `/login` (гость) | Ответ содержит `<footer>` |
| Копирайт | GET `/login` | footer содержит `© 2026 LFC.ru` |
| Дисклеймер | GET `/login` | footer содержит `Не является официальным сайтом Liverpool FC` |
| Гостевой auth блок | GET `/login` (без сессии) | HTML содержит ссылку `/login` и `/register` в nav |
| Авторизованный auth блок | GET `/` (с сессией) | HTML содержит `<form` с `action="/logout"` в nav |
| HTMX partial | GET `/login` + `HX-Request: true` | Ответ НЕ содержит `<header` и `<footer` |
| HTMX partial | GET `/register` + `HX-Request: true` | Ответ НЕ содержит `<header` и `<footer` |
| Skip-link target | GET `/login` | HTML содержит элемент с `id="content"` |
| Skip-link | GET `/login` | HTML содержит `<a href="#content"` |
| aria-label | GET `/login` | HTML содержит `aria-label="Основная навигация"` |
| Семантический main | GET `/login` | HTML содержит `<main` |

### E2E / визуальные тесты (браузер)

| Сценарий | Ожидаемый результат |
|---|---|
| Короткий контент (< viewport высота) | `footer.getBoundingClientRect().bottom ≈ window.innerHeight` |
| Длинный контент (> 2000px) | footer находится ниже последнего элемента контента |
| Tab на странице | Skip-link появляется при первом нажатии Tab |
| Клик «Форум» (заглушка) | Страница не меняется (href="#") |

## 12. Не входит в scope

- Адаптивное (responsive) меню с бургером для мобильных — отдельная задача
- Страницы разделов (Форум, Новости) — только ссылки-заглушки
- Смена темы (тёмная/светлая)
- Логотип в виде изображения — пока текст «LFC.ru»
- Breadcrumbs
- Active state навигации — добавляется при реализации соответствующих разделов
