# FT-005: Forum Structure

## What

### Problem

Форум существует как идея, но не реализован как функциональная единица. Структура «разделы → темы → сообщения» отсутствует полностью. Блок «Активность форума» на главной ссылается на несуществующие страницы.

### Outcome

| Metric ID | Metric | Baseline | Target | Measurement method |
|---|---|---|---|---|
| `MET-01` | Авторизованные пользователи, оставившие хотя бы один пост | 0 | Рост после запуска | `COUNT DISTINCT user_id FROM posts` |

### Scope

- `REQ-1` GET `/forum` отображает список разделов (название, описание, счётчик тем)
- `REQ-2` GET `/forum/sections/:id` отображает список тем раздела (название, автор, дата последнего поста, счётчик постов)
- `REQ-3` GET `/forum/topics/:id` отображает список постов темы (текст, имя автора, дата)
- `REQ-4` Авторизованный пользователь может создать пост в теме; неавторизованный — не может (кнопка «Ответить» скрыта)
- `REQ-5` Авторизованный пользователь может ответить на пост с цитированием; ответ отображается с цитатой
- `REQ-6` Модератор/Администратор может создать раздел (название, описание)
- `REQ-7` Модератор/Администратор может создать тему в разделе (название, выбор раздела)

### Non-Scope

- `NS-1` Редактирование и удаление постов
- `NS-2` Поиск по форуму
- `NS-3` Real-time обновления постов через SSE (WF-04 — отдельная фича)
- `NS-4` Лайки, реакции, рейтинг постов

### Constraints / Assumptions

- `CON-1` Иерархическая модель данных `sections → topics → posts` → [ADR-004](../../adr/ADR-004-forum-hierarchy-model.md)
- `CON-2` Создание разделов и тем ограничено ролями `Moderator` и `Admin` (роли из `domain/problem.md`)
- `CON-3` Блок «Активность форума» на главной (FT-003) ссылается на страницы форума — переходы должны работать корректно

## Verify

### Exit Criteria

- `EC-1` Все `REQ-*` покрыты passing `SC-*`
- `EC-2` `CON-2` проверен: пользователь без роли получает 403 при попытке создать раздел/тему
- `EC-3` Переходы из блока «Активность форума» на главной ведут на корректные страницы форума

### Acceptance Scenarios

- `SC-1` Гость открывает `/forum` → видит список разделов с названием, описанием, счётчиком тем
- `SC-2` Гость открывает `/forum/sections/:id` → видит список тем с названием, автором, датой последнего поста, счётчиком постов
- `SC-3` Гость открывает `/forum/topics/:id` → видит все посты с текстом, именем автора, датой
- `SC-4` Авторизованный пользователь создаёт пост → пост появляется в теме
- `SC-5` Авторизованный пользователь отвечает на пост → ответ отображается с цитируемым текстом
- `SC-6` Администратор создаёт раздел → раздел появляется в `/forum`
- `SC-7` Ссылки из блока «Активность форума» на главной ведут на корректные страницы форума
- `SC-8` Администратор создаёт тему в разделе → тема появляется в `/forum/sections/:id`

### Negative / Edge Cases

- `NEG-1` Гость пытается создать пост → 401 или редирект на `/login`; кнопка «Ответить» скрыта в HTML
- `NEG-2` Пользователь без роли `mod`/`admin` пытается создать раздел → 403
