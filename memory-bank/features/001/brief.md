# 01_auth_basic

## Brief
[GitHub Issue](https://github.com/Pegorino82/lfcru_forum/issues/1?reload=1)

## Проблема
Сейчас посетители форума не могут создавать личные аккаунты и писать сообщения.
Без аутентификации невозможно развернуть ролевую модель и модерацию контента. Задача является первым шагом к MVP форума — без неё невозможно реализовать ни одну другую пользовательскую функцию.

## Для кого
- Гость (незарегистрированный посетитель) — регистрация
- Пользователь (зарегистрированный аккаунт) — вход в систему

## Желаемый результат
- Гость может создать личный аккаунт и получить возможность создавать темы и писать сообщения в открытых разделах форума
- Посторонний не может войти в чужой аккаунт без знания пароля
- Система защищена от автоматизированного подбора пароля
- Пользователь остаётся залогиненным между визитами без повторного ввода пароля

## Acceptance Criteria
- Гость может зарегистрироваться с email, соответствующим формату `user@domain.tld` (синтаксическая проверка); подтверждение email через письмо на MVP не требуется
- Пользователь может войти и получить возможность создавать темы и писать сообщения в открытых разделах форума
- Невалидные данные (неверный пароль) отклоняются
- При более чем 5 неудачных попытках входа с одного IP за 60 секунд — блокировка на 10 минут
- Сессия пользователя действует не менее 30 дней с момента последней активности
