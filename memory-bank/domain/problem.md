# Project Problem Statement

Этот документ фиксирует общий продуктовый контекст проекта. Feature-документы должны ссылаться на него, а не переписывать один и тот же background каждый раз.

PRD, если он нужен, не заменяет этот документ, а уточняет отдельную инициативу относительно уже зафиксированного project-wide контекста.

## Boundary With PRD

- `domain/problem.md` — общий для всего проекта контекст: продукт, ключевые workflows, top-level outcomes и устойчивые ограничения.
- `prd/PRD-XXX-short-name.md` — инициативный слой: какая именно продуктовая проблема берется в работу сейчас, для каких пользователей и с каким scope.
- Если новый документ просто повторяет общий фон проекта и не вводит initiative-specific scope, PRD создавать не нужно.

## Product Context

LFC.ru — русскоязычный фан-сайт и форум болельщиков ФК «Ливерпуль». Проект состоит из двух частей:

- **Сайт**: главная, новости, трансферы, турнирная таблица, календарь матчей, архив статей.
- **Форум**: иерархическая структура «разделы → темы → сообщения». Гости могут читать открытые разделы; авторизованные пользователи — создавать темы и писать сообщения.

Роли пользователей (возрастающий уровень прав): Гость → Пользователь → Модератор → Администратор.

Аутентификация: регистрация по email/паролю, вход, выход, восстановление пароля.

## Core Workflows

- `WF-01` Чтение новостей и статей на сайте (доступно без авторизации)
- `WF-02` Участие в форуме: создание тем, публикация сообщений, ответы (требует авторизации)
- `WF-03` Аутентификация: регистрация по email/паролю, вход/выход, восстановление пароля
- `WF-04` Real-time обновления форума через SSE — новые сообщения появляются без перезагрузки страницы
- `WF-05` Модерация: управление разделами, темами и пользователями (роли Модератор/Администратор)

## Constraints

- `PCON-01` SQL только через параметризованные запросы (`$1`, `$2`) — `fmt.Sprintf` в SQL запрещён
- `PCON-02` CSRF-токен обязателен для всех POST/PUT/DELETE — реализован через middleware
- `PCON-03` Rate-limiting на `/login` против брутфорса — реализован через `ratelimit` пакет
- `PCON-04` Весь runtime запускается в Docker; Go-тулчейн на хосте не требуется и не предполагается
- `PCON-05` Интеграционные тесты подключаются к БД `lfcru_test`, а не к `lfcru` — предотвращает уничтожение данных разработки
