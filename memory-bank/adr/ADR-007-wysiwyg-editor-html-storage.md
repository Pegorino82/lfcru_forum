# ADR-007: WYSIWYG-редактор и HTML-хранение тела статьи

## Контекст

Статьи на сайте поддерживают форматирование, но редактор — plain-text textarea с Markdown. Задача FT-023 требует WYSIWYG-интерфейса: форматирование через тулбар по выделению, вставка изображений в произвольное место, подписи к изображениям и выравнивание текста. Стандартный Markdown не поддерживает выравнивание и inline-изображения с подписями в portable-виде. Нужно выбрать формат хранения тела статьи и библиотеку редактора.

## Драйверы решения

- Проект использует HTMX + Alpine.js — без React/Vue; редактор должен работать как vanilla JS или подключаться через bundler.
- Выравнивание текста не выражается в стандартном CommonMark — требует либо HTML, либо нестандартного синтаксиса плюс Go-расширения парсера.
- Изображения с подписями в Markdown требуют нестандартного синтаксиса (`::: figure ... :::`) или сырого HTML-блока — в обоих случаях нужна поддержка на уровне парсера.
- XSS-санитизация обязательна на бэкенде при любом HTML-хранении.
- ADR-005: изображения хранятся на файловой системе + Docker volume, upload-endpoint уже существует.

## Рассмотренные варианты

| Вариант | Плюсы | Минусы | Роль кандидата |
| --- | --- | --- | --- |
| **A — Markdown + кастомный синтаксис + Goldmark-расширения** | Plain-text читаемость; Markdown-файлы переносимы | Нужно расширять Go-парсер (alignment, figure) + WYSIWYG должен уметь читать/писать этот синтаксис — двойная сложность; синхронизация форматов хрупка | Не основной |
| **B — HTML из WYSIWYG (выбрано)** | Редактор генерирует HTML напрямую; выравнивание = CSS-класс; no Go-парсер нужен; рендеринг через `template.HTML(body)` | Потеря plain-text читаемости в БД; XSS-риск требует санитизации (bluemonday) | Основной кандидат |
| **C — JSON TipTap Document** | Максимальная гибкость; структурированный формат | Vendor lock-in на схему TipTap; нужен кастомный рендерер в Go; сложнее в дальнейшей поддержке | Не основной |

### Выбор библиотеки (при Варианте B)

| Библиотека | Плюсы | Минусы |
| --- | --- | --- |
| **TipTap** (headless, ProseMirror) | Все нужные расширения из коробки (Image, TextAlign, Link, Bold, Italic, Strike, Heading); HTML-вывод; активно поддерживается; vanilla JS через `@tiptap/core` | Требует npm-бандлера или ESM CDN; чуть крупнее bundle |
| **Quill** | Зрелый; CDN-подключение без бандлера; простая конфигурация | Развитие замедлено; кастомные блоки (image с caption) требуют самописных Blot-ов; меньше гибкости |
| **Editor.js** | Чистая блочная модель; image-блоки естественны | JSON-выход (несовместим с Вариантом B напрямую); нужен кастомный Go-рендерер |

## Решение

Предлагается **Вариант B**: хранить тело статьи как HTML-строку в поле `body` таблицы `articles`. Редактор — **TipTap** (vanilla JS, headless), подключается через ESM CDN или npm-бандлер. Тело статьи санитизируется библиотекой **bluemonday** на бэкенде при сохранении и проверяется при отображении через `template.HTML`.

Решение действует на весь цикл FT-023: формат хранения, upload-интеграцию с ADR-005-endpoint, Go-обработчик save и шаблоны отображения.

## Последствия

### Положительные

- Не нужно расширять Markdown-парсер: выравнивание и figure — CSS-классы в HTML.
- TipTap покрывает все acceptance criteria FT-023 готовыми расширениями.
- Рендеринг статьи = `{{ .Body | safeHTML }}` без дополнительной обработки.

### Отрицательные

- Существующие статьи, хранящиеся в Markdown, перестанут рендериться корректно — нужна миграция или двойной рендер-путь.
- HTML в БД нечитаем глазом без парсинга.
- Зависимость от bluemonday (Go) для XSS-санитизации.
- Bundle-зависимость TipTap: нужно решить вопрос сборки JS (npm + esbuild или ESM CDN).

### Нейтральные / организационные

- `memory-bank/domain/architecture.md` необходимо обновить: зафиксировать, что `articles.body` хранит HTML.
- Если в будущем потребуется экспорт статей в Markdown — нужна конвертация HTML→Markdown.

## Риски и mitigation

| Риск | Mitigation |
| --- | --- |
| XSS через user-supplied HTML | Санитизация через bluemonday с allowlist-политикой (разрешить только нужные теги и атрибуты) |
| Существующие Markdown-статьи сломаются | FT-023 явно исключает миграцию (NS-03); деградация существующих статей — ожидаемое поведение согласно ASM-03 до выполнения отдельной миграционной задачи. |
| TipTap CDN недоступен | Проверить локальный bundling как fallback; зафиксировать в PRE плана |
