# ADR-003: Email-only Registration Without Email Confirmation in MVP

## Контекст

При проектировании регистрации пользователей (FT-001) нужно выбрать метод аутентификации и определить, требуется ли подтверждение email для активации аккаунта. Проект на стадии MVP — время до первого публичного запуска критично.

## Драйверы решения

- Минимальное время до запуска MVP
- Фан-сайт с доверенной аудиторией — риски spam-регистраций ниже, чем в коммерческом продукте
- Нет инфраструктуры для отправки email (SMTP не настроен)
- Социальный вход добавляет OAuth-зависимости, выходящие за рамки MVP

## Рассмотренные варианты

| Вариант | Плюсы | Минусы | Статус |
|---|---|---|---|
| Email + пароль, без подтверждения | Быстрый старт, нет SMTP-инфры | Возможна регистрация с чужим email | **Выбрано** |
| Email + пароль + подтверждение email | Верификация владения email | Требует SMTP, усложняет UX | Отложено (пост-MVP) |
| Социальный вход (Google/VK) | Нет управления паролями | OAuth-зависимость, усложнение инфры | Отклонено для MVP |

## Решение

MVP регистрации использует только email и пароль. Синтаксическая проверка email (формат `user@domain.tld`) обязательна; подтверждение email через письмо — отложено. Социальная аутентификация не реализуется.

## Последствия

### Положительные

- Нет SMTP-инфраструктуры на MVP
- Быстрое прохождение регистрации для пользователя
- Простая реализация

### Отрицательные

- Пользователь может зарегистрироваться с чужим email (до добавления подтверждения)
- Нет гарантии владения email-адресом
- Восстановление пароля через email недоступно без SMTP (NS-3 из FT-001)

### Нейтральные / организационные

- `NS-1` из FT-001 явно фиксирует дефер подтверждения email
- Когда email confirmation будет добавлен — потребуется миграция существующих аккаунтов

## Риски и mitigation

| Риск | Mitigation |
|---|---|
| Регистрация с чужим email | Приемлемо для MVP на аудитории фан-сайта; пост-MVP — добавить подтверждение email |
| Спам-аккаунты | Модераторы могут блокировать аккаунты (WF-05 в `domain/problem.md`) |

## Follow-up

- Пост-MVP: реализовать подтверждение email через SMTP (создать ADR при выборе провайдера)
- Пост-MVP: восстановление пароля (NS-3 из FT-001)
